Hibernate知识点总结回顾

一、Hibernate的5大重要对象以及相关API

序号 对象 主要内容 重要关注点/方法
1 Configuration 加载配置文件 负责与数据库连接，生成sessionFactroy
2 SessionFactory 创建/获取session，二级缓存存储的地方 getCurrentSession()/openSession()
3 Session 负责与数据库交互CRUD的API对象 get()/load()的区别;sava/persist区别;saveOrUpate/merge区别
4 Transaction 事务对象
5 Query 封装查询的API对象
6 Criteria 更加面向对象的查询方式API对象

二、持久化对象的生命周期

生命周期的判断依据

  1. 是否有OID
  2. 是否被session管理
状态 描述 特点
临时状态/瞬时态(transient) 刚刚用new语句创建，没有被持久化，不处于session中 没有oid,不在session当中
持久化状态(persistent) 已经被持久化，加入到session的缓存中。 有oid,在session当中
脱管态/游离状态(detached) 已经被持久化，但不处于session中。 有oid,不在session当中
删除状态(removed) 对象有关联的ID，并且在Session管理下，但是已经计划被删除。 有oid,在session当中,最终的效果是被删除

session负责改变状态,事务负责同步数据.

三、对象的关系映射

3.1 对象的关系分类：

1. 依赖关系
2. **关联关系**
3. 聚合关系
4. **组合关系**
5. 泛化关系

理解：定义这些关系的目的是为了减少代码，减少重复编写(引用即可)

3.2 关联关系映射

3.2.1 单向关联

小结：

  1. 看关系配置在哪一方，该方就负责维护关系。
  2. 不要轻易认为代码上employee.setDep(xxx)就建立了关系，(1)假如部门没保存，其OID是不确定的(取决于主键的生成策略)，所以关系仍然没在数据库中建立。(2)还需要实体的保存顺序
  3. 默认都使用延迟加载，many可以采用if-null判断是因为拥有外键
3.2.2 双向关联

重点：inverse=true设置在one放提高性能

总结:

  1. 在映射一对多的双向关联关系时,应该在one方把//元素的inverse属性设置为true,提高性能.
  2. 在建立两个对象的双向关联时,应该同时修改两端对象的相应属性:
    dept.getEmps().add(emp);
    emp.setDept(dept);
  3. 当删除双向关联的关系时,也应该修改两端对象的相应属性:
    dept.getEmps().remove(emp);
    emp.setDept(null);

五、Hibernate的缓存

1. 一级缓存：session级别缓存

2. 二级缓存：应用application级别的缓存(意思就是应用重启之后才失效->栗子：Tomcat重启失效)，借助于第三方框架的缓存(EHACHE)

六、OpenSessionInView

通过过滤器处理，把session绑定在请求中，最后结束时才关闭session。
之前session的关闭是在事务提交后，现在延迟到请求结束后一步。
