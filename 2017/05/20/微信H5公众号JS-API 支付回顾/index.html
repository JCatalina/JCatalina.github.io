2. 微信支付流程图

  • 我们要开发的实际上是 4-9 的步骤。

3. 步骤开发

(一)总体发起支付步骤JS页面代码

  1. 判断是否是UserAgent是否是微信浏览器
  2. 是微信浏览器，通过post请求我们服务器向微信服务器下单。
  3. 下单成功后，post请求成功返回JSAPI调用微信支付的参数config，并配置在页面JS。
  4. 调用callPay,微信浏览器发起支付->用户输入密码。

2. 流程图第六步：根据下单接口返回的数据，生成JSAPI页面调用支付的参数和签名，并返回给H5页面。

4. 过程中遇到的问题

  • 问题1. 用户唤醒了支付，想着跟商家再交流下价格，然后取消了支付，后来价格改了之后重新发起支付，但是下单失败。
  • 分析：因为微信下单之后，微信服务器已经保存了++订单号和价格++，加入此时你重新下单同样的订单号，但是价格不一样了，微信是不能下单成功的。因此
  • 解决：下单的时候，发送给微信服务器的++订单号=我们系统订单号+”时间戳”++，这样可以避免下单失败，然后微信回调通知我们支付成功时，返回的订单号也是++订单号=我们系统订单号+”时间戳”++我们系统在++提取系统订单号++做逻辑即可.
  • 问题2. 微信支付如何测试
  • 解决：
  1. 在微信公众号内增加配置测试支付的授权目录为测试服的域名
  2. 把自己生成在正式公众号的微信的openid写死在下单函数里面测试。(因为微信公众号绑定了微信商家，微信支付要求发起支付的用户的openid一定是要与之关联的微信公众号下的openid)。
    • 走过的弯路：在测试服时，虽然微信商家那里配置了测试服的支付授权域名，但是下单时采用的是测试公众号下的微信用户openid，所以下单不成功。

5.过程中额外的收获知识点。

  • 关于证书，签名和签名算法，是大学密码学的学的内容。
  • 非对称加密的两个应用：加密和数字签名。
  • 微信大概就是用这些方法来防止其他服务器代理发送假的支付消息等等安全问题。
